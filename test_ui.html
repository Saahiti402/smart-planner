agent_service.py

Travel Planning Agent

Tests travel_planning_agent(trip_data) — orchestrates RAG context retrieval + user preferences + itinerary generation. This is the full pipeline. Requires login.

Step 1 — Save User Preferences (optional)

The agent reads saved preferences from DB to personalize the trip. Requires JWT token.

Step 2 — Run Agent: Plan Trip

Uses /plan-trip which internally calls travel_planning_agent(). The agent selects tools, fetches RAG context, loads preferences, and builds the itinerary.
